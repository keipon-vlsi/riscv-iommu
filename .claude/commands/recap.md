# /recap — 前回作業の復習

引数: $ARGUMENTS (省略可。"3d" "week" 等で範囲指定可能。省略時は最新 1 日分)

## Phase 1: ログファイル特定

1. `doc/worklog/` 配下の `.md` ファイルを Glob で列挙
2. **$ARGUMENTS の解釈**:
    - 省略 → 最新 1 ファイル
    - `3d` / `3days` → 最新 3 ファイル
    - `week` / `7d` → 過去 7 日分
    - `YYYY-MM-DD` → その日付のファイル
    - その他 → 「どの範囲にしますか?」と AskUserQuestion

## Phase 2: ログ内容の読み込み

対象ファイル全てを `Read` で読み込む。各ファイルの以下を抽出:

- Session トピック一覧
- Done で達成したこと
- Open questions / TODO (未完了のもの)
- Next session の指示

## Phase 3: 要約を表示

以下のフォーマットで 画面表示 (ファイル生成は不要):

```
================================================================
📋 直近の作業状況 (<対象期間>)
================================================================

## 最近やったこと

<日付順に 3-5 行でまとめ。動詞で始めて簡潔に>
- <date>: <要約>
- <date>: <要約>

## ファイル変更サマリ

<主要な創作物・修正物を列挙>
- <path> (<状態>)

## 未解決の TODO

- [ ] <日付> - <TODO>
- [ ] <日付> - <TODO>

## 前回 "Next session" で予告していた内容

<最新ログの Next session をそのまま引用>

================================================================
💡 今日は何から始めますか?
================================================================
```

## Phase 4: ルール

- **ログを改変しない**: recap は読むだけ。書き換え禁止
- **推測しない**: ログに書かれていないことは「ログに記述なし」と明記
- **古い TODO の状態確認**: 可能なら新しいログで完了したか追跡し、重複表示しない
- **短くまとめる**: recap が長くなりすぎないよう、各項目 1-2 行に収める
